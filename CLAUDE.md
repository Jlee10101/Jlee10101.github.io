## Running locally

This project is now set up for macOS. There is no Node build step or package manager; use the included shell launcher, which serves the folder with macOS' system Python 3.

- **Start:** double-click [`start-server.command`](start-server.command) in Finder, or run `./start-server.command` from Terminal. The script auto-opens `http://localhost:8080/`.
- **Stop:** Ctrl+C in the Terminal window.
- **Port:** `8080` is hardcoded in [`start-server.command`](start-server.command) - change `PORT` if conflicts arise.

Keep the launcher ASCII-clean and executable so Finder can run it directly.

Opening `.html` files by double-click (file://) mostly works but will break the logo image path in the nav (spaces in `/assets/images/logo/...` only resolve correctly under HTTP). Always use the server.

## Page architecture

Three page types, in the user's intended flow:

1. **[`index.html`](index.html)** — landing. Centered `Make Better.` logo with two paper-fold panels in corners and a "Click to enter" cue. Whole stage is one `<a>` wrapping the logo, pointing to `work.html`.
2. **[`work.html`](work.html)** — the 2×2 project grid (Heinrich Zaunschirm-style: image, uppercase title, one-line tagline, year marker). Also contains the About and Contact bands, anchored via `#about` and `#contact` — nav links on project pages point here with those hashes.
3. **[`projects/*.html`](projects/)** — one file per project (`atonal`, `innova`, `bloomroom`, `skechers`). Image-book projects are long vertical scrolls of full-width `<img>` tags; Skechers embeds `assets/pdfs/skechers-book.pdf`. Prev/next links at the bottom cycle through all four projects circularly.

There is deliberately no SPA routing. Every navigation is a full page load, softened by the `is-leaving` fade transition wired up in [`assets/script.js`](assets/script.js).

## Generating a new project detail page

The image-book project pages are dense (26–49 `<img>` tags each) and tedious to edit by hand. They were generated by a bash loop that (a) naturally-sorts the filenames with `sort -V` and (b) URL-encodes spaces (`%20`) and plus signs (`%2B`) in the `src` attributes — essential because asset folders have names like `INNOVA Project Core 4` and filenames like `LEE JAKE INNOVA Process Book + Presentation.jpg`. If adding a 5th project:

1. Drop the images into `assets/images/<Folder Name>/` (the filenames must end `<prefix>.jpg`, `<prefix>2.jpg`, `<prefix>3.jpg`, ... for `sort -V` to order them correctly).
2. Regenerate the `.html` by reusing the `build_project` shell function pattern (see session history). Don't hand-type the `<img>` list.
3. Add a `grid__item` tile to [`work.html`](work.html) and update the prev/next circular links on all project pages so the new slug is part of the loop.

## Styling system

Everything is in [`assets/styles.css`](assets/styles.css) — single file, no preprocessor. Notable choices:

- **Design tokens** (root custom properties): `--ink` `#0d1016`, `--paper` `#f5f3ed`, `--blue` `#1e2ad1` (the brand Klein blue from the logo). Do not introduce ad-hoc colors — extend the token set instead.
- **Type stack:** `--f-display` is `"Nordique Pro", "Archivo", ...`. Nordique Pro is a commercial typeface; the `.otf` in `assets/` is a *trial/partial* version and may render with missing glyphs for numerals/punctuation. **Archivo (Google Fonts) is the working fallback** and carries the full character set — all display type is designed to read acceptably in either face. Do not source Nordique Pro from unauthorized URLs; if the user obtains a license, they'll drop the full OTFs into `assets/` with the same filenames.
- **Body classes** scope page-specific rules: `.landing`, `.work-page`, `.project-page`. When adding a new kind of page, follow the same pattern rather than adding free-floating selectors.
- **The custom dot cursor was removed** as off-brand. Do not reintroduce `cursor: none` or a `.cursor` element — the user rejected that specifically.

## Conventions

- The hero logo PNG and paper-fold JPGs live under `assets/images/logo/` and `assets/images/Paper folds/` respectively. Spaces in folder names are intentional and must be `%20`-encoded in every `href`/`src`.
- The four project slugs are permanent: `atonal`, `innova`, `bloomroom`, `skechers`. Prev/next nav, page titles, and grid tile `href`s all reference them.
- `loading="lazy"` is set on every project book `<img>`. Keep it — the Atonal book alone is ~50 images totaling ~100 MB, and eager loading blocks the page.
